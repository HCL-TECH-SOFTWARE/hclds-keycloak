# Configuration Parameters

The following document provides details on available configuration parameters to steer the behavior of the Keycloak deployment. All properties are defined in the included `values.yaml` file and can be overwritten with custom values to fit the needs of your environment.

On a high level, the deployment can be adjusted regarding the following areas via respective Helm parameters:
- Global - images
- Common - naming, dns
- Keycloak - ssl, admin, spi, env vars
- Statefulset - ports, replicas, lifecycle, init containers
- Exposure - external networking, ingress
- RBAC - rules, service accounts
- Metrics - monitoring
- Config CLI - enable and preconfigure
- Database
- Cache
- And more

All paramters can be found at [https://github.com/bitnami/charts/tree/main/bitnami/keycloak#parameters](https://github.com/bitnami/charts/tree/main/bitnami/keycloak#parameters), and are also listed [below](#configuration-parameters).

## Applying configuration parameters

There are three notable ways for applying / providing the parameters to your helm deployment

### Via a custom `values.yaml` file

You can overwrite individual properties from the [values.yaml](../../../deployment/helm/hclds-keycloak/values.yaml) file that's used by default. To do so, you can create a `custom-values.yaml` file with your desired configuration properties:

```sh
vi custom-values.yaml
```

```yaml
# enable production mode
production: true

# production mode requires TLS to be set to true
tls:
  enabled: true
  autoGenerated: true

ingress:
  enabled: true
  ingressClassName: "nginx"
  hostname: "<YOUR_HOSTNAME>"
  tls: true
  selfSigned: true
```

The above configuration is an example that would enable the production mode and configure TLS with an auto-generated SSL certificate, as well as configure it to use an existing ingress service to be exposed through.

Finally, provide the file as part of the `helm upgrade` command:

```sh
helm upgrade -f custom-values.yaml keycloak .
```

### By setting them individually within the `helm upgrade` command

Each property can be set via the `--set` argument as part of the `helm upgrade` command:

```
helm upgrade keycloak .
    --set image.repository=keycloak 
    --set service.type=NodePort
```

### By editing an existing deployment

Lastly, changes can also be applied to existing deployments by editing them. Note that not all parts here are accessible. If you want to adjust properties on the keycloak stateful set, run the following command:

```
kubectl edit sts keycloak
```

Then find the appropriate value and edit and save it. If the change is valid, Kubernetes will apply them and restart the service to make them take effect.
